博客搭建日记1

引言

本博客使用Nodejs+Hexo搭建而成
主题使用的是一款简单的卡片式UI设计主题Butterfly
博客托管在码云
不知不觉，我的博客已经改动很多内容了，回想这一次次的修改，我觉得有必要详细记录一下博客搭建的过程，以防我不小心搞崩了博客…

Hexo简介

Hexo是一款基于Node.js的静态博客框架，依赖少易于安装使用，可以方便的生成静态网页托管在GitHub和Coding上，是搭建博客的首选框架。

搭建博客优点:

  • 超快速度: Node.js 所带来的超快生成速度，让上百个页面在几秒内瞬间完成渲染
  • 一键部署: 只需一条指令即可部署到 GitHub Pages, Heroku 或其他平台，无需服务器与域名
  • 支持Markdown: 只要写md文件，就能生产blog
  • 插件和可扩展性： 超级多的插件，无所不能。留言板，聊天，阅读量统计，url短链接转化等等

环境配置

  1. 安装Git

安装好后，用git --version查看版本,本机Git版本v2.33.1.windows.1
顺便说一下，windows在git安装完后，就可以直接使用Git Bash来敲命令行了，不用自带的cmd。（Git Bash的复制ctrl+ins;粘贴shift+ins）

  1. Hexo是基于nodeJS编写的，所以需要安装一下NodeJs和里面的npm工具

安装好后，用node -v和npm -v检查一下有没有安装成功并查看版本,本机Nodejs版本v16.13.0,npm版本v8.1.0。
（npm安装好后需要设置下npm在安装全局模块时的路径和环境变量，防止安装模块的时把模块装到C盘，占用C盘的空间，或安装好hexo后无法使用）

  1. 安装Hexo

前面Git和Nodejs安装好后，就可以安装Hexo了，你可以先创建一个文件夹Blog，然后到这个文件夹下Git Bash。
输入命令npm install -g hexo-cli安装

hexo常用基本指令

为了方便，每次准备推送时,可以hexo c && hexo g && hexo s和hexo c && hexo g && hexo d

博客部署到Github

以上环境配置好后，在Blog文件夹输入 hexo init 命令初始化博客，完成后，Blog文件夹目录下有：

  • node_modules: 依赖包
  • public：存放生成的页面
  • scaffolds：生成文章的一些模板
  • source：用来存放你的文章
  • themes：主题
  • _config.yml: 博客的配置文件**

等文件
然后hexo c && hexo g && hexo s后，就可以在浏览器输入localhost:4000就可以看到生成的博客了

在Github创建一个和你用户名相同的仓库，后面加.github.io，只有这样，将来要部署到GitHub page的时候，才会被识别，也就是xxxx.github.io
建议建立一gh-pages分支，并作为默认分支，以后生成的页面都将部署到该分支

然后将Hexo和GitHub关联起来，也就是将Hexo生成的页面部署到GitHub上，打开站点配置文件 _config.yml

这个时候需要先安装deploy-git ，也就是部署的命令,这样你才能用命令部署到GitHub。

然后hexo c && hexo g && hexo d，过一会儿就可以在http://yourname.github.io这个网站看到你的博客了！
部署带Github经常会有超时错误且网站响应慢，部署到Gitee步骤类似

注意： 部署到Gitee要勾选强制使用HTTPS

关于域名

如果部署到Gitee Pages不可以自定义域名

部署到Github，进入自己的域名控制台添加一条CNAME类型的解析记录

并在Github Pages页面上Custom domain处填写你的域名

！填上域名

为避免每次上传都要输入可在Blog目录source文件夹下建一CNAME文件，在里面输入域名
